CSS Syntax: Selectors

When you want to style a section of content in a web page, you need to figure out how to reference that content. In CSS terms, this is called a selector.

Tag selectors

Any HTML tag can be a selector. Furthermore, you can create custom selectors to suit your own needs. A tag selector uses just the tag you're selecting without any modifications. This example will make all h1 elements twice as big.

h1 {font-size: 200%;}

Class selectors

In addition to selecting a specific tag you can select an element that you've given a "nickname" to in the HTML using a class attribute. Class selectors are essentially custom selectors you create, give a name to, and then apply to any element you want.

For example, what if you wanted a few of your images to have a border around them but not all of them. You could do this easily by applying a class to the ones you want to look this way:

<img src="example.jpg" class="fancy">

Style rules that are targeting a class name are different than those that target a specific element. Class selectors start with a . before the class name to indicate you're ignoring the element itself and looking at the element's class attribute instead. Here's what it looks like:

.fancy {border: 5px solid white;}

Descendent selectors

What if we wanted to set the font size of all paragraphs contained within the footer of your page but not in any other area? The best way to do this is to use another type of selector: a descendent selector.

A descendent selector tells the browser to look in only the areas of your page, then to only apply the style to a certain tag/id/class inside it. For example, the following code would cause paragraphs to be smaller but only if they're inside the foorter:

footer p {font-size: 75%;}

Multiple selectors

When it is necessary to apply a style to multiple content divisions, you can easily combine selectors by separating them with commas. Here's an example where text in both paragraphs and lists are changed to the Arial font:

p, li {font-family: arial;}

This would be the same thing as writing two separate style rules, one for paragraphs and another for list items. Adding both selectors separated by a comma allows us to talk to both elements at the same time and simplify our CSS.
